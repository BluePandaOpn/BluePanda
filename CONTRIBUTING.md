# Contributing to BluePanda

Thanks for helping improve BluePanda.

## Workflow

1. Fork the repository.
2. Create a feature branch from `main`.
3. Keep changes focused and documented.
4. Add or update docs for any API change.
5. Open a Pull Request with clear summary.

## Pull Request checklist

- [ ] Code runs locally.
- [ ] Public API changes are documented in `docs/`.
- [ ] `CHANGELOG.md` includes relevant entry under `Unreleased` or target version.
- [ ] Licensing headers/notices remain intact.

## Style

- Prefer readable and simple Python.
- Avoid hidden behavior.
- Keep backwards compatibility when possible.

## Commits

Suggested style:

- `feat: add ...`
- `fix: resolve ...`
- `docs: update ...`
- `refactor: simplify ...`

## Reporting bugs

Include:

- BluePanda version,
- Python version,
- OS,
- minimal reproduction snippet,
- expected vs actual behavior.
