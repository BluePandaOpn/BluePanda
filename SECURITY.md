# Security Policy

## Supported version

- `v0.5.x` (current)

## Reporting a vulnerability

Please report security issues privately to the maintainer before public disclosure.

When reporting, include:

- affected version,
- vulnerability type,
- proof of concept,
- impact assessment,
- suggested mitigation (if available).

## Disclosure process

1. Report received and acknowledged.
2. Issue reproduced and triaged.
3. Fix prepared and validated.
4. Public disclosure with patch notes in `CHANGELOG.md`.
